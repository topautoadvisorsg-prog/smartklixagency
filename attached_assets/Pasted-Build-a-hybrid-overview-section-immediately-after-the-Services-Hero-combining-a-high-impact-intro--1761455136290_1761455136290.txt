🎨 Design & Layout
Element	Description
Layout	Two-tier structure — top = centered text block; bottom = 4 equal cards in a grid (2×2 on tablet, 1×4 stacked on mobile).
Background	Light gradient (#FFFFFF → #F9FAFB).
Card Style	Soft rounded corners (20 px), shadow (rgba(0, 0, 0, 0.06)), even padding (~40 px), white background.
Typography	Poppins Bold (headlines), Inter Regular (body).
Text Colors	Headings: navy #0D1B2A; body: medium gray #4F4F4F; accents: gold #F4B400.
Icons	Lucide-react line icons (gold, 32–36 px, top-center).
Spacing	~160 px top / 140 px bottom; 60 px between intro and card grid.
Alignment	Everything centered and evenly spaced across screen width (max 1200 px container).
✨ Motion & Interaction (Bring It to Life)
Element	Animation / Effect
Section Entry	Smooth fade-in and slide-up (trigger via IntersectionObserver).
Headline Glow	Subtle animated gradient shimmer across “Smart Systems” text (CSS background-clip animation).
Cards (on Scroll)	Staggered entry — 0.15 s delay per card (fade + upward motion).
Cards (Hover)	Scale 1.02× + gold shadow glow (box-shadow: 0 0 20px rgba(244, 180, 0, 0.3)).
Icons (Idle)	Gentle pulse every 6 seconds (keyframes).
Icons (Hover)	Quick 5° tilt + brighter gold fill transition (0.4 s).
Divider (Optional)	Thin gold connecting line that subtly animates left-to-right when scrolled into view.
Mobile Behavior	Reduced motion — fade only, no scale or parallax.
⚙️ Developer Implementation Notes

Keep all animations pure CSS or native IntersectionObserver — no heavy libraries.

Use Lucide-react icons only (no SVGs).

Maintain SmartKlix spacing, color, and typography system from the homepage.

Ensure fully responsive design — smooth stack and spacing.

Optimize for performance (use will-change: transform, opacity on animated elements).

Add light gradient particle background (optional subtle moving dots) for extra depth.
